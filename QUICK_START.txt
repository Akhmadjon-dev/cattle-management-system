================================================================================
📊 WHAT BACKEND SERVES
================================================================================

GET /cattle
  → List cattle (with filters: status, breed, search)

GET /cattle/:id
  → Single cattle

POST /cattle
  → Create cattle (requires: tag, breed, gender, birth_date)

PATCH /cattle/:id
  → Update cattle

DELETE /cattle/:id
  → Soft delete (status → 'removed')

GET /cattle/:id/health-events
  → Health events for cattle

POST /cattle/:id/health-events
  → Create health event

GET /analytics ✨ NEW TIER 1 FEATURES
  → All metrics including:
    - healthCoverage (% with records)
    - alerts (action items sorted by severity)
    - removalStats (mortality rate)
    - breedPerformance (rankings)

💭 KEY TALKING POINTS
================================================================================

On Features:
  "Full CRUD operations, health tracking, and Tier 1 analytics that go
   beyond basic reporting. Alerts, health coverage %, mortality rate,
   breed performance—things farmers actually use to make decisions."

On Architecture:
  "Shared TypeScript types keep client/server in sync. Express + SQLite
   backend (simple, scales). React frontend with bookmarkable URLs.
   Clean separation throughout."

On Tier 1 Analytics:
  "Spec left analytics choice open. I asked: what does a farmer need?
   Not reports—action items. So I added alerts for cattle needing
   attention, health coverage % to spot neglect, mortality rate for
   health, breed rankings for data-driven breeding."

On Time Management:
  "Finished in 3.5 hours (within scope). Prioritized features over tests.
   Would add Jest + Vitest, migrations, structured logging in production."

🚨 IF SOMETHING BREAKS
================================================================================

Frontend won't start:
  → npm run build (check for TypeScript errors)
  → Clear browser cache (Ctrl+Shift+Delete)
  → Restart: npm run dev

Backend won't start:
  → Check port 3001 isn't in use: netstat -an | grep 3001
  → Delete cattle.db and restart (resets database)

Types not updating:
  → TypeScript compiled from shared/type.ts
  → Run: npm run build

Analytics shows zeros:
  → Create some cattle first (go to home page)
  → Add some health events
